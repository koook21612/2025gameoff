# Unity 项目协作规范

## Git 提交规范：约定式提交
https://www.conventionalcommits.org/zh-hans/v1.0.0/

## Git 协作流程

为了高效协作并避免冲突，请遵循以下流程：

### 1. 开始工作前
```bash
# 确保在主分支并获取最新代码
git checkout main
git pull origin main

# 创建新功能分支（推荐）
git checkout -b feature_你的功能名
# 或者复用已有的功能分支
git checkout 已有的分支名
```

### 2. 开发与提交
- 在分支上进行开发工作
- 完成一定进度后提交更改：
```bash
git add .
git commit -m "描述：清晰说明所做的修改"
```

### 3. 推送代码
```bash
git push origin 你的分支名
```

### 4. 请求合并
- **在微信中通知**："我推送了 [分支名]，功能已完成，请合并"
- 等待负责人合并到主分支

### 5. 开始下一轮工作
- 切换回主分支并拉取最新代码：
```bash
git checkout main
git pull origin main
```
- 重复步骤1，创建新的功能分支

## 注意
- ✅ **务必**在功能分支上开发，不要直接在main分支修改
- ✅ **务必**在开始新工作前拉取最新main分支
- ❌ 不要在未拉取更新的main分支上创建新分支

## 场景协作规范

### 新系统开发
- 每个主要功能系统创建独立的测试场景
- 在 `Scenes/Systems/` 目录下创建
- 开发阶段在自己的系统场景中工作

### 主场景修改
- 修改主场景前必须在群里沟通
- 尽量保持主场景的稳定性
- 优先考虑通过预制体和脚本扩展，而非直接编辑主场景

### 场景合并策略
- 系统完成后，由负责人将其集成到主场景
- 集成时选择人少的时间段操作
- 如有冲突，立即沟通解决
